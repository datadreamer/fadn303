Servers, Clients and Protocols

Aaron Siegel | | Spring 2016

The Network

  • The network is comprised of computers, routers, and cables connecting them.
  • Some of the computers are servers and some of them are clients.
  • Servers and clients are deliniated by their purpose.

Servers and Services

  • Servers are computers that run services for clients.
  • Services are software applications to perform tasks.
  • Clients connect to servers on different ports to access different services.
  • Multiple clients can make use of services simultaneously.

Clients

  • Clients are computers that run software to connect to services on servers.
  • Clients issue requests/commands to servers.
  • Clients = users, visitors, consumers, customers.

Protocols

  • Services use unique protocols to allow servers and clients to communicate.
  • These communications occur between ports on the server and client computers.
  • HTTP operates on port 80 of the server.
  • FTP operates on port 21 of the server.
  • SFTP operates on port 2222 of the server.

HTTP

  • Hypertext Transfer Protocol (c. 1991).
  • Specifies how information and files are transferred on the web.
  • Used by web servers and web browsers.

HTTP Client: Web Browser

  • Firefox, Chrome, Safari, Internet Explorer, etc.
  • Browser downloads files and renders the web page in your window.
  • How is accessing a site using Safari different from doing so with Firefox?
  • How is accessing a site using Safari on your iPhone different from doing so on your MacBook?
  • How is navigation different on your site when using a touch screen?
  • How does your site present information on displays with different aspect ratios?

FTP

  • File Transfer Protocol (c. 1971).
  • Specifically for moving files from one system to another.
  • Used by FTP servers and FTP clients.

FTP Client: FileZilla

  • Free, open-source FTP client.
  • Allows transfer of files between your computer and an FTP server.
